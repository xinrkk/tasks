## 🌐学习内容

### :one: GitHub的作用  
- GitHub 是一个基于 Git 的 **代码托管平台**，开发者可以在上面:   
    - 管理代码版本
    - 进行团队协作
    - 发布开源项目
    - 创建个人仓库


### :two: GitHub的入门操作  

- 创建新仓库  
    - 创建新仓库  
    - 填写项目名称、描述  
    - 选择公开（Public）或私有（Private）  
    - 创建新文件或上传文件  

- Git基本命令  
    - git pull:从远程仓库拉取最新代码，并与本地分支合并。  
    - git push:将本地分支的最新提交推送到远程仓库。  
    - git commit:将暂存区（git add 添加的内容）提交到本地仓库。  
    - git add:把修改的文件添加到“暂存区”，为提交（commit）做准备。  
    - git clone:从远程仓库复制整个项目到本地。  

- GitHub名词解释 
    - | 名称 | 含义 | 备注 |
      |:----:|------|------|
       | **Repository（仓库）** | 存放项目代码的地方 | 类似文件夹 |
       | **Commit（提交）** | 保存修改的记录 | 记录每次更新 |
       | **Branch（分支）** | 代码的不同版本线 | 可用于开发新功能 |
       | **Merge（合并）** | 把分支内容合并到主分支 | 常见于 PR 流程 |
       | **Pull Request（PR）** | 请求合并分支的操作 | 开源协作核心机制 |
       | **Fork（派生）** | 复制他人仓库到自己账户 | 开源贡献常用 |
       | **Star（星标）** | 收藏仓库 | 便于日后查找 |

### :three:如何在GitHub中获取资料
- 搜索
	- 在 GitHub 顶部直接搜索框输入关键词
	- 使用 Topics（主题标签）精准查找
    - 阅读项目的 README.md 获取详细说明
    - 查看 [Trending 热门项目榜单](https://github.com/trending)

- 获取资料
    - 直接下载 ZIP
        - 打开仓库页面  
        - 点击 **“Code” → “Download ZIP”**  
        - 解压即可使用
    - 使用 git clone
        - 将整个项目复制到本地，方便查看和修改。

- 收藏与关注
    - Star：收藏项目  
    - Fork：复制项目到自己账户  
    - Watch：关注项目更新
